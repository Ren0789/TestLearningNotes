2. 安装WebDriverAgent（全手动配置）

https://github.com/appium/WebDriverAgent/tree/6e1617c02f00f750a7d89ad76f7e1e1e53e968a9

1.配置驱动

1. 找到appium的安装路径：/usr/local/lib/node_modules/appium/,进入node_modules/_appium-xcuitest-driver@2.54.0@appium-xcuitest-driver/WebDriverAgent
2. 打开终端并转到该位置，然后运行以下命令以设置项目
    mkdir -p Resources/WebDriverAgent.bundle
    ./Scripts/bootstrap.sh -d
3. 在步骤一的文件夹中选择WebDriverAgent.xcodeproj并使用xcode打开
4. 打开WebDriverAgentLib和WebDriverAgentRunner
    ![](https://github.com/imurchie/appium-xcuitest-driver/blob/isaac-rs/docs/xcode-config.png)
5. General-->选中‘Automatically manage signing’-->选择在准备过程中准备的个人开发者账号-->自动选择Signing Ceritificate，结果如上图
6. Xcode可能无法为WebDriverAgentRunner 目标创建配置文件，需要进入Build Settings-->Product Bundle Identifier-->将‘com.facebook.WebDriverAgentRunner’修改，我的修改为‘com.zhangxiaochuan.WebDriverAgentRunner’
7. 然后回到命令行，执行xcodebuild -project WebDriverAgent.xcodeproj -scheme WebDriverAgentRunner -destination 'id=<udid>' test

2.问题解决

1. 回到appium中执行inspector报错error Domain=NSCocoaErrorDomain Code=260 "The file “WebDriverAgentRunner-Runner.app” couldn’t be opened because there is no such file." userInfo={NSFilePath=/Users/zhangbingwei/Library/Developer/Xcode/DerivedData/WebDriverAgent-dikkwtrisltbeobjmfvpthwwekvs/Build/Products/Debug-iphoneos/WebDriverAgentRunner-Runner.app, NSUnderlyingError=0x7feed31246b0 {Error Domain=NSPOSIXErrorDomain Code=2 "No such file or directory"}}
    解决方案：
    1. 在xcode中执行product-->scheme-->WebDriverAgentRunner 以及 product-->destination-->你的设备
    2. product-->test
    3. 进入工程目录查找WebDriverAgentRunner-Runner.app文件
    4. 将其复制到提示文件不存在的路径中
    5. 重新启动appium的inspector
